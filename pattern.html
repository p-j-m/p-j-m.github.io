패턴
패턴은 사용자가 목적을 달성하기 위한 과정입니다. 컴포넌트로 만들어진 페이지로 구성되며 사용자 경험의 흐름을 고려해야합니다. 마음에 드는 글을 저장하거나 다음 여행지의 호텔을 예약하는 것과 같이 복합적인 목적을 달성하는 과정을 설계하는 것으로 사용자가 각 단계를 예측할 수 있어야 합니다.
요소
서비스의 핵심 목적을 달성하기 위해 다양한 패턴을 사용합니다. 복잡하게 조합이 되어 구성되므로 종류를 구분하기가 어렵지만 어떤 패턴에서도 3가지 요소가 필요합니다.
  • 맥락: 사용자가 어떤 상황에서 무엇이 필요한가
  • 흐름: 어떤 과정을 거치는가
  • 구현: 어떻게 인지하고 사용하는가
고르기
3가지 요소를 구체적으로 이해했다면 되도록 OS에서 제시하는 보편적인 패턴을 사용하는 것이 사용자 경험을 유지하는데 도움이 됩니다. (사용자는 언제나 OS를 사용하던 중 서비스를 이용할 것입니다.)

만들기
서비스에 알맞게 기존의 패턴을 변형하거나 이전에 경험하지 못한 패턴을 제시해야 할 때가 있습니다. 먼저 맥락을 파악한 뒤, 다른 서비스 혹은 현실에서 어떤 패턴으로 문제를 해결하는지 조사합니다. 이후 사용자의 행동 단계에 맞춰 구현합니다. 이해하기 쉽도록 많이 사용되고 있는 검색 패턴을 예시로 살펴보겠습니다.
검색
사용자가 가지고 있는 단서로 원하는 것을 찾는 기능입니다. 사용자가 찾고자 하는 정보 (사용자에게 필요한지 몰랐던 정보조차)를 찾을 수 있게 해주는 것이 목적입니다.
  • 맥락: 사용자가 기억하는 단어로 영화 찾고 재생하기
  • 흐름: 단어 입력 - 결과 확인 - 더 자세히 보기
  • 형태: 텍스트 필드, 그리드 리스트
맥락 짜기
사용자가 속해 있는 상황을 정리합니다. 육하원칙에 따라 정리하는 것이 편리하며 웹, 모바일과 같이 환경이 제한적일 경우(스마트폰이나 모니터로 서비스를 이용하는 환경)에는 다른 원칙보다 "무엇을 어떻게"에 집중합니다. 이 과정을 통해 '목적'을 정의합니다.
흐름 짜기
맥락에서 찾아낸 "목적"을 달성하기 사용자가 어떤 단계로 접근하는지를 정리합니다. 목적을 달성하기 위한 단계별 "단계별 목적"을 정의하며 이것을 기준으로 페이지를 디자인합니다.
경우의 수
흐름에서 찾은 "단계별 목적"에 맞춰 발생할 수 있는 경우의 수를 예측합니다. 서비스에서는 사용자의 행동에 의해 많은 경우의 수가 발생합니다. 단계별로 어떤 변수가 있는지 찾아내고 예측되는 결과를 정리합니다.
알고리즘 짜기(선택적)
가능하다면 '경우의 수' 단계에서 얻은 결과를 토대로 어떤 구조로 작동할지를 예측합니다.
형태 짜기
"흐름"과 "경우의 수"를 기준으로 페이지를 제작합니다. 기준이 되는 화면 사이즈에서 영역별로 달성해야할 단계별 목적을 지정합니다. 해당 영역에 단계별 목적을 달성할 수 있는 적합한 컴포넌트를 적용한뒤 변수를 고려하여 제작합니다.
  • 단어 입력: 타 플랫폼 및 서비스에서 반복적으로 사용되는 컴포넌트로 상단 고정
  • 결과 확인: 최대한 많은 결과를 보여주기 위해 넓은 영역 지정
  • 선택 후 재생: 포스터와 제목 만으로도 영화를 선택하고 재생할 수 있지만 더 자세한 내용을 확인하고 재생하는 경우를 위해 최대한 많은 정보를 줄 수 있도록 별도 페이지 제작
레이아웃 짜기
형태에 대한 개념이 정리가 된 뒤에는 시각적인 컴포넌트를 배치합니다. 배치만으로도 GUI 디자인이 완성됩니다.
